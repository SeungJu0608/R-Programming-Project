0.1분위수가 2+3x를 만족하는 데이터셋을 다음과 같이 생성하였다.

set.seed(1)
n =1000
X2 = rnorm(n)

beta.true = c(2,3)
x_beta = beta.true[1] + beta.true[2] * X2

Y2 = rnorm(n=n, mean=x_beta+1.3)

원래 선형회귀에서 오차의 정규성과 독립성, 그리고 등분산성이 만족되었을때 Y가 정규분포를 따르고 이러한 오리지널 선형회귀를 떠올려 보았을때 추정된 회귀식은 각 x값에 대한 반응변수 y의 기대값을 구하는 모형이다. 이는 quantile(0.5) 즉 특정 x에 대한 y의 mean값을 추정하는 것과 같다. 또한 각 x 값에서의 quantile을 대표하는 y들을 따라 선을 그었을때 그 기울기가 3으로 모두 동등할 것이다. (오차의 가정때문) 따라서 원 데이터가 2+3x를 따르는 Y의 분포에 일정 상수만큼 Y축 평행이동 시켜주어 특정 x값에 대한 하위 10% y들을 이었을때 2+3x를 따르는 데이터를 생성하였다.

### (f) 부트스트랩은 반복적인 적합을 통해 변동성을 지닌 모형의 계수에 대한 standard error를 구하고자 쓰는 기법이다. 이때 상사가 이야기하는 방법에 대해서 생각해 보았을때 오점을 전산실험의 결과 시각화한 그래프를 통해 알 수 있다. 직관적으로 생각해 보았을 때 먼저 부트스트랩 데이터 셋 1000개를 통해 적합한 회귀 직선과 yhat값들은 선형회귀를 통한 적합값으로 이들 각각은 E(Y|X=x)를 구한것이다. 이들을 나열하여 10% 값에 해당하는 데이터들은 각각의 X데이터(설명변수)에 대한 조건부 Y의 기댓값의 변동성에 대한 하위 10%를 구한것이다. 그러나 실질적으로 quantile(0.1)은 특정 X값에 대한 변수 Y의 전체 확률 분포에서 하위 10%를 이야기한다. 따라서 상사의 얘기에 반박할 수 있다.**

set.seed(1)
n =100
x.ex = rnorm(n)

beta.true = c(2,3)
x_beta = beta.true[1] + beta.true[2] * x.ex

y.ex = rnorm(n=n, mean=x_beta+1.3)

mat = data.frame(x.ex, y.ex)
mat.tr = mat[1:80,]
mat.val = mat[81:100,]

pred.ex = matrix(rep(NA,1000*20), nrow=20, ncol=1000)
ord.ex = t(pred.ex)

for (b in 1:1000) {
  set.seed(b*10)
  ind.tr.b = sample(1:80, size =80 , replace = T)
  tr.b = mat.tr[ind.tr.b,]
  fit.b = lm(y.ex~x.ex,data=tr.b)
  pred.ex[,b] = predict(fit.b, newdata=mat.val)
}

for (i in 1:nrow(pred.ex)) {
  ord.ex[,i] = sort(pred.ex[i,])
}

percent.ex= ord.ex[100,]

fit.r = rq(y.ex~x.ex, data= mat.tr, tau=0.1)
pred.r = predict(fit.r, newdata = mat.val)

ori.linear = lm(y.ex~x.ex, data= mat.tr)
ori.pred = predict(ori.linear,newdata = mat.val)
